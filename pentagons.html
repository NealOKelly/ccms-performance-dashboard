Batch schedule

About the data

This page shows data collected supplied by Capgemini as part of their daily checks. These data a generated at part of the 'Report Set' which is run overnight. They are manually uploaded to Google Sheets. As such, the charts below may not reflect the current network performance.

Note that Capgemini do not supply daily check data on non-working days.

The date supplied is the Report Date. The average runtime figures reported relate to the 24 hour period immediate prior to the Report Set being run.

What does it measure?

The batch schedule orchestrates batch processing in CCMS. There are a number of batch processing jobs that are run continuously (on loop) within CCMS. These are:

  • Continuous Background Set 1
  • Continuous Background Set 2
  • Continuous Background Set 3
  • Provider Synchronisation

These data show the average runtime of each of those jobs.

Why does it matter?

The performance of the batch schedule can be thought of as a proxy indicator for overall system performance. Longer runtimes might mean a degradation in performance (or — alternatively — an increase in workload).

Although, these are background processes, there may be direct end user impacts of an increase in runtimes — providers and caseworkers may have to wait longer for system processes to complete before they can move on to their next task.
